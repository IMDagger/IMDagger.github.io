Не до конца ясно для меня почему так, но скорее всего это связано как-то с рефлексией. Создание переменной типа procedure и присваивание в неё ссылки на метод класса приводит к тому, что дальнейшие её вызовы медленнее, чем аналогичная переменная, которая содержит lambda выражение с явным вызовом требуемого метода. Т.е. вот такой код (скомпилированный компилятором Kawa под целевую платформу JVM) исполняется на моём Intel Atom D525 с 1.8 GHz где-то не более 0.8 секунды:

(module-export Test)

(define-simple-class Test ()
    (internal-field ::procedure init: (lambda () ((this):real-method)))
    ;(internal-field ::procedure init: (this):real-method)
    ((real-method) ::int (+ 1 2))
    ((iter (MAX ::int)) :: void
        (let loop ((i ::int 0))
                ((this):internal-field)
                (when (< i MAX) (loop (+ i 1))))))

((Test):iter 10000000)

А вот такой код исполняется уже около 8 секунд: (вариант с set! внутри конструктора работает аналогично)

(module-export Test)

(define-simple-class Test ()
    ;(internal-field ::procedure init: (lambda () ((this):real-method)))
    (internal-field ::procedure init: (this):real-method)
    ((real-method) ::int (+ 1 2))
    ((iter (MAX ::int)) :: void
        (let loop ((i ::int 0))
                ((this):internal-field)
                (when (< i MAX) (loop (+ i 1))))))

((Test):iter 10000000)

Скалярная хранимая процедура

Сегодня спроецировал процедуру, которая возвращает число 1 или 0 на скалярную величину типа boolean. Вышло не очень сложно, но был подводный камень. Писать нужно не Boolean, а boolean, т.е. тип из разряда встроенных, но Hibernate вернёт объект в списке типа Boolean:

<?xml version="1.0"?>
<!DOCTYPE hibernate-mapping PUBLIC
    "-//Hibernate/Hibernate Mapping DTD 3.0//EN"
    "http://hibernate.sourceforge.net/hibernate-mapping-3.0.dtd\ ">
<hibernate-mapping>
<sql-query name="users.isCanReadLog" callable="true">
<return-scalar column="isYouCanShow" type="boolean" />
{ call spIsCanShowStrangerLog(:courseId, :userId) }
</sql-query>
</hibernate-mapping>

Сначала я посмотрел пример, но там было написано {? = call spIsCanShowStrangerLog(:courseId, :userId) }, но эта запись вызывает ошибку, т.к. я передаю два параметра, а оно хочет 3-и. Всё дело в том, что пример не для MSSQL, для MSSQL не нужно передавать результат в параметр. Работать нужно из DAO с данным запросом так:

List result = getHibernateTemplate().
        findByNamedQueryAndNamedParam("users.isCanReadLog",
                             new String[]{"courseId", "userId"},
                             new Object[]{courseId, userId}
        );
logger.info((Boolean) result.get(0));

К встроенному boolean тип Object не конвертируется, а Hibernate возвращает всё типа Object.
